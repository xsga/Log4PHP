Differences from Apache

A true modern fork,
not just a version bump

Xsga Log4PHP modernizes architecture, typing, tooling, and observability readiness for current production systems.

Area Apache Log4PHP Xsga Log4PHP
PHP Version >= 5.2.7 PHP 8.4+
Codebase Legacy patterns Strict types, namespaces, PSR-4
Code Quality Not enforced PSR-12 + Psalm 6 + lint tooling
PSR-3 No Full LoggerInterface support
Structured Logging Limited LoggerLayoutJson + context-rich logs
Appender Strategy Broad legacy list Curated modern appenders + Loki
Core Features

Built for high-throughput services
and clean developer workflows

Fast enough for production traffic, simple enough for day-one adoption.

HP

High Performance, Low Overhead

Efficient logging internals focused on minimizing runtime cost in API and microservice workloads.

JS

Structured Logging (JSON)

Emit ingestion-ready JSON events for centralized search, analytics, and cloud observability pipelines.

LV

Complete Log Levels

Supports ALL, TRACE, DEBUG, INFO, NOTICE, WARNING, ERROR, CRITICAL, ALERT, EMERGENCY, and OFF.

AP

Modern Appenders

File, DailyFile, RollingFile, Console, and native Loki appender for production-grade pipelines.

CF

Flexible Configuration

Configure via XML or PHP arrays with thresholds, layouts, and root logger controls.

DX

Easy Integration

Composer-first setup and PSR-3 compatibility make integration straightforward in existing codebases.

log4php.xml 
<configuration threshold="all">
  <appender name="json" class="LoggerAppenderRollingFile">
    <param name="file" value="logs/app.json.log" />
    <layout class="LoggerLayoutJson" />
  </appender>
</configuration>
